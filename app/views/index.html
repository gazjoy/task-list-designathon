Task lists

Progress focussed task list

Rationale

The purpose of tags in a task list are to provide meaning to the overall service progress and to understand what is the status of a specific task, has it been started, completed or not? This example doesn't use the tag component, I'd like to explore different ways of providing the progress information.

There are 4 statuses each with nuanced behaviours:

  • [In progress] - I have started this one so enable me to complete it. Have at the top.
  • [Not started] - If I get stuck on a task I might want to jump onto another task meaning I'd have multiple tasks in progress. Have it at the top if nothing is in progress.
  • [Cannot start yet] - If I cant start it, I don’t need to interact with it, have it lower down the page. Move it to [Not started] and let me interact with it when I can start it.
  • [Completed] - I am done with it, keep it on the page and let me check my answers but have it at the bottom. The check link would go to a check your answers page with anwers from the specific task.

As a user moves through their tasks, the tasks move from [Not started], to [In progress] and [Completed]. The end state before submission of all data for the service would be all tasks are [Completed]. All other statuses would not show on the task list screen.

Considering screen reader users

Putting tasks in this order would give screen reader users the most important task to work on first rather then having them navigate through completed tasks before getting to the next one to do.

Examples of the same service as a user progresses through it.

  • Progress focussed – no tasks started
  • Progress focussed – tasks in progress
  • Progress focussed – all tasks completed
